◆ AceLevel

enum class CBJCards::AceLevel : int
strong

The AceLevel enum represents the contextual value of an ace.

In blackjack, an ace can have a value of either 1 or 11. These values are represented by ACE_LOW and ACE_HIGH, respectively. NOT_AN_ACE is provided so that the AceLevel enum can be used to represent any card, not just aces.

The reason this context is given in a separate enum class is so that the Rank enum class can remain simple and unpolluted by values like ACE_LOW and ACE_HIGH.

Enumerator
NOT_AN_ACE 

The value of a non-ace card.

ACE_LOW 

The value of an ace when it is counted as 1.

ACE_HIGH 

The value of an ace when it is counted as 11.

◆ Rank

enum class CBJCards::Rank : int
strong

The Rank enum represents the rank of a card.

This enum class assigns integer values to the ranks of a standard deck of playing cards. The value of ACE is set to 1, and subsequent ranks are assigned incrementally. This allows for easy comparison of ranks so that cards have an order.

Additionally, this scheme means that ranks from TWO to TEN can always be cast to an integer to get their numeric value.

Enumerator
ACE 

The rank of an ace, equivalent to 1. Represented by the string A when supplied to the Card class's static function rankToSymbol. Aces can have a numeric value of either 1 or 11, depending on the context.

enum class CBJCards::Suit : int

The Suit enum represents the suit of a card.

This enum class arbitrarily assigns integer values to the suits of a standard deck of playing cards. By associating integer values to each Suit value, it becomes possible to compare the suits so that cards have an order.

Values are as follows:

Enumerator
HEARTS 

The suit of hearts, equivalent to 0. Represented by the ♥ character when supplied to the static function Card::suitToSymbol.

DIAMONDS 

The suit of diamonds, equivalent to 1. Represented by the ♦ character when supplied to the static function Card::suitToSymbol.

CLUBS
